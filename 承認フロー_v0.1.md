# アンソロウ 承認フロー v0.1（厳守原則）

> **目的**: システム変更を「動作確認 → 承認 → 保存／改善 → 適用」で必ず通すこと
> **狙い**: 一流データセンター級の信頼性 ＋ 樋口さんが哲学的問いに集中できる時間

## フロー全体図

```
┌─────────────────────────────────────────────────────────┐
│ ① 変更提案 (人間 or AI)                                  │
│    変更前 / 変更後 / 理由 / 影響範囲                       │
└────────────────────────┬────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│ ② 動作確認 (樋口さんがブラウザ／実機で確認)                  │
└────────────────────────┬────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│ ③ 樋口承認 — 必ず2択を選択                               │
│   ┌─────────────────┐    ┌─────────────────────────┐    │
│   │ A. 保存する     │    │ B. 変更しない           │    │
│   │   (= 適用)      │    │   (= 改善内容を確認)     │    │
│   └─────────────────┘    └─────────────────────────┘    │
└────────────────────────┬────────────────────────────────┘
        ┌────────────────┴────────────────┐
        ↓                                 ↓
┌──────────────────────┐      ┌──────────────────────────┐
│ A: 適用 → 書き換え   │      │ B: 改善内容を提示 →      │
│   versions/v0.x/に   │      │   ①に戻る                │
│   スナップショット   │      │                          │
└──────────────────────┘      └──────────────────────────┘
```

## 自動承認の対象（低リスクのみ）

| 種別 | 条件 | 自動可否 |
|---|---|---|
| 値引き稟議 | 3%以下 ／ 粗利率15%以上維持 | ◯ |
| 提案テンプレ微修正 | 過去30日成約データ準拠 | ◯ |
| 自顧客タグ更新 | 担当本人の操作 | ◯ |
| 値引き稟議 | 5%超 | ✗ 必ず人手 |
| 契約条件変更 | 期日／特約／金額 | ✗ 必ず人手 |
| 権限付与 | 新規ユーザー／アクセス拡張 | ✗ 必ず人手 |
| AIプロンプト改訂 | バージョンアップを伴う | ✗ 必ず人手 |

**原則**: プロジェクト実行に必要 かつ 低リスクなら自動承認。それ以外は必ず樋口承認。

## バージョン管理（厳守3版）

```
versions/
├── v0.1/   ← 初版（現在稼働中）
├── v0.2/   ← 次版（承認次第ここに保存）
└── v0.3/   ← 安定版
```

**保存トリガ**: 樋口承認直後、自動でスナップショット → `versions/v0.x/` 配下に格納。

**復元**: バージョン履歴画面（DB-04）から1クリックで切替可能。差分プレビューで影響範囲を必ず確認。

## エラー時の挙動

- プログラムがエラー状態 → **再生成** を自動実行
- 3回連続エラー → 樋口に通知 ＋ 直近成功版に復元
- 復元後も状態を `versions/エラー/` に保管（再現可能性のため）

## 記録する内容（一度生成した内容は記録）

| 記録対象 | 保存先 |
|---|---|
| 全画面HTML | `current/prototypes/` |
| デザイントークン | `current/design-system/tokens.css` |
| 仕様書 | `current/specs/` |
| 承認履歴 | `versions/v0.x/承認ログ.md` |
| 差分 | `versions/v0.x/diff/` |

このフローはアンソロウ全システムで一貫適用されます（UserView / Marketplace / Dashboard）。
