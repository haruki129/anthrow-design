# Dashboard APP 仕様書 v0.1

> **対象**: 管理者・マネージャー（部長／本部）が組織を俯瞰する場
> **思想**: 「**見るべき数字を、見るべき順番で**」— 個別ドリルダウン可能・承認・履歴

## 画面一覧（4画面）

| # | 画面 | 役割 | ファイル |
|---|---|---|---|
| 01 | KPIサマリー | 月次・週次の主要KPIを俯瞰 | `dashboard/01_KPI.html` |
| 02 | 案件パイプライン | 全案件のステージ別カンバン | `dashboard/02_パイプライン.html` |
| 03 | 承認センター | 全承認案件の集中管理 | `dashboard/03_承認.html` |
| 04 | バージョン履歴 | システム変更3版保存・復元 | `dashboard/04_バージョン履歴.html` |

## 主要メトリクス

- **粗利率** / **成約率** / **客単価** / **稟議所要時間（中央値）**
- **担当者別案件数** / **エリア別需要分布**
- **AI推薦命中率**（提案カード→成約に至った率）

## 承認ワークフロー（厳守原則）

```
変更提案 → 動作確認 → 樋口承認 →
  ├ 保存 ─→ versions/v0.x/ にスナップショット
  └ 改善 ─→ 改善内容を確認 → 再提案
```

## v0.1 → v0.2 改善候補

- 異常検知（成約率急落・追客滞留など）の自動アラート
- 担当者間のチーム分析（属性別の成約パターン抽出）
- 経営層向けエグゼクティブビュー（極限まで情報を絞った1画面サマリー）
